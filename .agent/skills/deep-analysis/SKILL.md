# Deep Analysis Skill

Activate enhanced analytical thinking for complex challenges requiring comprehensive reasoning.

## When to Use

- Complex architectural decisions
- Strategic technology choices
- Multi-stakeholder problems
- Trade-off analysis
- Systems thinking challenges

## Analysis Framework

### 1. Parse the Problem
- Extract the core challenge
- Identify stakeholders and constraints
- Recognize implicit requirements
- Question assumptions
- Surface unknowns

### 2. Multi-Dimensional Analysis

#### Technical Perspective
- Technical feasibility and constraints
- Scalability, performance, maintainability
- Security implications
- Technical debt and future-proofing

#### Business Perspective
- Business value and ROI
- Time-to-market pressures
- Competitive advantages
- Risk vs. reward trade-offs

#### User Perspective
- User needs and pain points
- Usability and accessibility
- User experience implications
- Edge cases and user journeys

#### System Perspective
- System-wide impacts
- Integration points
- Dependencies and coupling
- Emergent behaviors

### 3. Generate Multiple Solutions

Brainstorm 3-5 different approaches. For each:
- Pros and cons
- Implementation complexity
- Resource requirements
- Potential risks
- Long-term implications

Include conventional AND creative solutions.

### 4. Deep Dive Analysis

For promising solutions:
- Detailed implementation plans
- Pitfalls and mitigation strategies
- Phased approaches and MVPs
- Second and third-order effects
- Failure modes and recovery

### 5. Cross-Domain Thinking
- Parallels from other industries
- Design patterns from different contexts
- Biological or natural system analogies
- Innovative combinations

### 6. Challenge and Refine
- Devil's advocate each solution
- Identify weaknesses and blind spots
- "What if" scenarios
- Stress-test assumptions
- Unintended consequences

### 7. Synthesize Insights
- Combine insights from all perspectives
- Identify key decision factors
- Highlight critical trade-offs
- Present nuanced view

## Output Structure

```markdown
## Problem Analysis
- Core challenge
- Key constraints
- Critical success factors

## Solution Options

### Option 1: [Name]
- Description
- Pros/Cons
- Implementation approach
- Risk assessment

### Option 2: [Name]
[Similar structure]

## Recommendation
- Recommended approach
- Rationale
- Implementation roadmap
- Success metrics
- Risk mitigation plan

## Alternative Perspectives
- Contrarian view
- Future considerations
- Areas for further research
```

## Key Thinking Principles

1. **First Principles**: Break down to fundamental truths
2. **Systems Thinking**: Consider interconnections and feedback loops
3. **Probabilistic Thinking**: Work with uncertainties and ranges
4. **Inversion**: Consider what to avoid, not just what to do
5. **Second-Order Thinking**: Consider consequences of consequences

## Example Use Cases

```
# Architectural decision
Should we migrate to microservices or improve our monolith?

# Complex problem solving
How do we scale our system to handle 10x traffic while reducing costs?

# Strategic planning
What technology stack should we choose for our next-gen platform?

# Design challenge
How can we improve our API to be more developer-friendly 
while maintaining backward compatibility?
```

## Output Expectations

- Comprehensive analysis (2-4 pages of insights)
- Multiple viable solutions with trade-offs
- Clear reasoning chains
- Acknowledgment of uncertainties
- Actionable recommendations
- Novel insights or perspectives

## Meta-Analysis

After completing analysis:
- Reflect on the thinking process
- Identify areas of uncertainty
- Acknowledge biases or limitations
- Suggest additional expertise needed
- Provide confidence levels for recommendations
