# Frontend Polish Skill

Transform frontend into pixel-perfect, industry-standard implementation.

## When to Use

- Improving UI consistency and polish
- Auditing design systems
- Fixing alignment and spacing issues
- Enhancing mobile responsiveness
- Improving accessibility compliance

## Enhancement Areas

### 1. Pixel-Perfect Alignment & Spacing
- **Grid System Audit**: Ensure 8px grid system consistency
- **Alignment Verification**: Check all elements are perfectly aligned
- **Spacing Consistency**: Eliminate random margins/padding
- **Visual Hierarchy**: Establish clear visual rhythm

### 2. Color System & Consistency
- **Color Palette Audit**: Reduce to systematic palette
- **Contrast Validation**: WCAG AA compliance (4.5:1 for text)
- **Color Harmony**: Fix mismatched colors
- **Semantic Colors**: Proper primary, secondary, success, warning, error

### 3. Typography Standards
- **Font Size System**: Typographic scale (base 16px)
- **Font Family Consistency**: Limit to 2-3 families
- **Line Height Optimization**: 1.5-1.7 for body text
- **Font Weight Hierarchy**: Clear hierarchy (400, 500, 600, 700)

### 4. Component Enhancement
- **Header/Footer**: Professional, feature-rich designs
- **Button Consistency**: Standardized styles and states
- **Form Enhancement**: Proper validation states
- **Card System**: Consistent shadows and borders

### 5. Mobile Responsiveness
- **Breakpoints**: mobile 320px+, tablet 768px+, desktop 1024px+
- **Touch Targets**: Minimum 44px for mobile
- **Mobile-First**: Optimize for mobile first
- **Fluid Typography**: Use clamp() functions

### 6. Visual Polish & Micro-interactions
- **Hover States**: Meaningful effects for interactive elements
- **Focus Indicators**: Clear, accessible focus states
- **Loading States**: Skeleton loaders and smooth transitions
- **Error States**: Well-designed error messages

### 7. Performance & Accessibility
- **Image Optimization**: WebP, lazy loading, responsive images
- **Semantic HTML**: Proper HTML5 elements
- **ARIA Labels**: Where needed for accessibility
- **Keyboard Navigation**: Full keyboard accessibility

## Implementation Checklist

- [ ] Scan for inconsistent spacing/alignment

### Phase 3: Component Enhancement
- [ ] Redesign header with navigation, search, user menu
- [ ] Enhance footer layout
- [ ] Standardize button components
- [ ] Improve form designs

### Phase 4: Responsive Implementation
- [ ] Implement mobile-first responsive design
- [ ] Optimize touch targets
- [ ] Test across device sizes

### Phase 5: Polish & Optimization
- [ ] Add micro-interactions and transitions
- [ ] Implement loading and error states
- [ ] Optimize performance
